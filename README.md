# ポケ抽選メモ

日本国内のポケモンカードゲーム抽選情報をJSONで整理し、応募状況、当選発表、購入期限をブラウザだけで管理するMVPです。

> **非公式サービスです。** 本アプリは株式会社ポケモン、任天堂、クリーチャーズ、ゲームフリークとは一切関係ありません。実際の応募条件・日時は必ず販売店の公式情報源で確認してください。

## 主な機能

- 「今すべきこと」を優先するダッシュボード、抽選一覧・詳細・応募履歴
- 応募ページから戻った時の完了確認（自動で応募済みにしません）
- 月表示・締切順スケジュール、単一／抽選単位／表示月のICS出力（Asia/Tokyo）
- `pokeca-lottery-1.0` JSONのZod検証、snake_case変換、差分プレビュー、承認インポート
- 安定した`offerKey`と段階的マッチング。更新時も抽選IDと応募状態を維持
- 外部AIチャット用の安全な調査プロンプト生成（AI APIは不使用）
- 表示地域、オンライン限定、非公式情報、confidence、テーマ設定
- IndexedDB全データのJSONバックアップ／確認付き復元／確認文字列付き削除
- 開発環境限定の架空サンプルデータ
- Web App Manifest（Service WorkerによるオフラインキャッシュはMVP対象外）

## 技術構成

Next.js App Router / React / TypeScript strict / Tailwind CSS / Dexie.js (IndexedDB) / Zod / date-fns / date-fns-tz / Lucide / Vitest / Testing Library。認証、サーバーDB、有料API、スクレイピングはありません。

## セットアップ

Node.js 20以降とpnpmを用意してください。

```bash
pnpm install
pnpm dev
```

`http://localhost:3000`を開きます。テスト、lint、production build：

```bash
pnpm test
pnpm lint
pnpm build
pnpm start
```

## データ保存とバックアップ

商品、販売店、抽選、応募状態、インポート履歴、設定は端末のIndexedDBに保存します。テーマの小さな設定だけlocalStorageにも保存します。パスワード、Cookie、認証トークン、カード情報は保存しません。

ブラウザデータ削除、プライベートモード終了、端末故障などでデータが消える可能性があります。「設定」から永続ストレージを要求し、定期的に「全データJSONエクスポート」を保存してください。復元は現在の全データを置換するため、確認画面を表示します。

## JSONインポート

1. 「調査」で条件を指定し、プロンプトをコピー／保存します。
2. ChatGPT等の外部AIチャットで実行します（Web検索可否は利用サービスによります）。
3. JSONのみの回答を「取込」へ貼り付けるかファイル選択します。
4. 検証結果と新規・更新・変更なし・競合を確認し、承認した項目だけ確定します。
5. 情報源URL、日時、引用根拠を必ず人が確認します。

未知の`package_version`は拒否します。既存データは自動削除せず、類似候補は自動マージしません。締切更新で元抽選と判定できる場合はIDを維持し、`UserLotteryState`も残します。

## 外部AI用プロンプト

アプリ内にAI APIやAPIキーはありません。「調査」で期間・商品・販売店・地域・件数を指定します。生成プロンプトには公式優先、推測禁止、不明値`null`、プロンプトインジェクション無視、JSON限定、既存データの簡略版が含まれます。

## サンプルデータ

開発サーバーでのみ「設定」に投入ボタンが表示されます。販売店・商品・URLはすべて架空／`example.com`で、受付中、本日締切、近日開始、結果待ち、購入期間、終了、条件不明、非公式を確認できます。実在の抽選はハードコードしていません。

## 既知の制限

- 通知、バックグラウンド同期、認証、端末間同期、応募自動化はありません。
- AI回答や外部ページを自動取得・検証しません。ログイン限定情報も推測しません。
- Manifestのみの軽量PWA対応です。Service Workerは更新・ビルド安定性を優先して未実装です。
- カレンダーはMVPの簡易月表示で、ICS受取側の解釈差があり得ます。
- ブラウザのストレージ制限に依存します。
- 販売情報機能は将来追加予定ですが、現在は抽選管理だけです。
